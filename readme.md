本项目主要是基于清华大学发布的知识库(http://keg.cs.tsinghua.edu.cn/project/ChineseKB) 构建属于我们自己的百科人物的
知识库，以便于后期利用其进行人物关联分析和可视化分析引擎。

1.项目运行环境：
Java 1.7，Eclipse以及64位的windows操作系统
中科院发布的中文分词工具ICTCLIS
Neo4j数据库

2.构建流程说明
(1)下载清华大学发布的知识库文件(.owl格式)，并将其导入到neo4j数据库中(关系数据库)，本文使用的Jena对owl格式的数据库文件进行
解析，并将解析后的内容加入到neo4j数据库中，构成我们的本地初始知识库。实现见JenaOWL.java

(2)利用百科知识库的内容对本地知识库进行扩充。首先使用爬虫程序对百科网页进行爬取，然后利用Jsoup对爬取的网页内容进行解析，
得到百科页面中infoBox里面的内容和人物关系页面以及摘要信息，将infoBox和人物关系页面的内容存储成三元组的形式（SP0），实现见
BaikeFunction.java,Hudong.java以及NetSpider.java

(3)将爬取的内容与知识库中的内容进行融合，若知识库中不存在要加入的实体同名的实体，则直接将其作为新的节点加入知识库，同时将其
百科页面的属性和关系加到其节点上。若知识库中存在于要加入实体同名的实体（可能存在多个），主要是利用相似度比较的方法，利用百科
人物页面的摘要信息与本地知识库中实体的描述信息和实体类别信息进行相似度计算，选择相似度大于一定阀值的且最高的实体作为要加入实体
在本地知识库中对应的实体（简单的实体链接方法，更加具体的方法见GCCEL和CCEL），然后将其信息也加入到该对应的实体节点上，详细的实体
过程见代码NefHelper.java

(4)系统的主函数见Connection.java
